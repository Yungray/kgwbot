핵심 가설

  • 반복적이거나 과거 사례 기반으로 대응 가능한 CS는 AI가 1차로 도와줄 수 있다. 유사 사례 검색, 처리 이력 요약, 답변 초안 생성을 보조합니다.
  • 운영자는 답변을 새로 만드는 대신 검토와 판단에 집중할 수 있다. 완전 자동화가 아니라 휴먼 피드백을 전제로 한 운영 보조 도구입니다.
  • 누적 CS 데이터는 기능 개선 우선순위를 잡는 근거가 될 수 있다. 반복 문의, 요청 상태, 기간별 처리 현황을 정리해 데이터 기반 제품 개발로 연결합니다.

향후 고민 중인 방향

CS 관리

능동적 개입

  • CS 인입 시 담당자 알림 문의가 들어온 순간 놓치지 않도록 알려줍니다.
  • 신뢰도 기반 자동 답변 일정 점수 이하는 담당자 컨펌 후 답변합니다.
  • 댓글 승인 알림 사용자 승인, 확인 댓글을 놓치는 일을 줄입니다.
배포 관리

ITGC 통제 판단

  • 통제 대상 여부 1차 판단 “이 건 ITGC 따라야 하나?”를 빠르게 확인합니다.
  • 판단 근거 제공 변경 범위와 영향도를 바탕으로 검토 포인트를 정리합니다.
VOC 기반 기능 개선

데이터 기반 개발 문화

  • 개선 우선순위 정리 반복 문의와 불편 포인트를 누적 데이터로 확인합니다.
  • 개선 근거 마련 기능 개선을 감이 아닌 실제 VOC 기반으로 설명합니다.

현재 리스크와 운영 방식

검색 성능이 정말 충분한가?

내가 원하는 답변이 나오는가?

답변 초안이 실제 운영자의 판단 방향과 맞는지 계속 확인해야 합니다.
