| 문서 버전 | v0.3 |

---

## 1. 배경

사내 아지트(Agit)에는 전자결재·인사 시스템 운영팀이 사용자로부터 받은 CS 문의·VOC가 글/댓글 형태로 누적되어 있다. 운영팀은 새 문의를 받을 때마다 과거 사례를 수기로 검색하고, 답변 패턴을 일관되게 유지하기 위해 상당한 시간을 쓰고 있다. 또한 누적된 VOC를 정기적으로 정리해서 제품팀에 전달하는 작업도 수작업으로 이루어진다.

## 2. 제품 목적

운영팀이 사내 아지트의 과거 사례를 빠르게 검색하고, 일관된 응대 톤으로 답변 초안을 생성하며, 누적된 VOC를 자동으로 통계·리포트화할 수 있게 한다.

## 3. 사용자

| 구분 | 정의 | 사용 빈도 |
|---|---|---|
| **Primary** | 전자결재·인사 시스템 운영팀 (CS 응대 담당) | 일 단위 (문의 발생 시) |
| **Secondary** | PM / 제품팀 (VOC 인사이트 소비) | 주·월 단위 (정기 리포트) |

## 4. 핵심 가치

1. **답변 작성 시간 단축** — 과거 사례 수기 검색 → 자동 추천, 답변 초안까지 LLM이 작성
2. **응대 일관성** — 운영팀 누구든 동일한 사례·톤으로 응대 가능
3. **검증 가능성** — 모든 답변 초안에 근거 사례 링크 + 처리 상태 메타 노출
4. **모듈 독립성** — 전자결재팀과 인사팀의 데이터·세션·이력이 완전 분리되어 동시 사용 가능
5. **운영 가시성** — 도구 호출, 토큰 사용, ID별 분포 등이 답변에 함께 표시되어 운영팀이 신뢰도 검증 가능

## 5. 범위 (In/Out Scope)

### In Scope
- 모듈 단위(전자결재, 인사 시스템) 챗봇 인터페이스
- 두 가지 작업 모드: **답변 가이드** (CS 응대 초안), **VOC 리포트** (통계·이슈 클러스터링)
- Agit Search API (`/v2/search/total`) 기반 글·댓글 검색
- Gemini 함수 호출(function calling)로 자동 도구 호출
- 모듈별 OAuth 토큰 분리 (전자결재/인사 각각)
- 이미지 첨부 (CS 스크린샷 OCR 분석 후 검색 키워드 추출)
- 답변 결과를 HTML 리포트로 다운로드

### Out of Scope (현재 버전)
- 모바일 전용 UI (1440 데스크톱 우선)
- 다중 사용자 관리 (현재는 단일 운영자용)
- 세션의 서버측 영속화 (in-memory dict, 재시작 시 소실)
- 모듈 동적 추가 (현재는 코드 `GROUPS` dict로 정의)
- LLM 응답 스트리밍

## 6. 성공 지표

| 지표 | 목표 | 측정 방법 |
|---|---|---|
| 답변 초안 작성 평균 시간 | 10초 이내 (Lite 모델, 캐시 미적용 기준) | 클라이언트 timer + 서버 log |
| 답변에 인용된 근거 사례 수 | 평균 3건 이상 | `find_similar_cases` 응답의 `top_cases` 길이 |
| 모듈 전환 시 데이터 누수 0건 | 100% 분리 보장 | `PINNED_GROUP` enforce + per-module localStorage 검증 |
| 도구 호출 실패율 (스키마 위반) | < 5% | tool_config=VALIDATED 적용 후 회귀 모니터링 |

## 7. 제약/가정

- **단일 운영자 환경**: 세션은 메모리 dict에 저장. 다중 인스턴스 / 멀티 워커 운영 시 별도 외부 저장 필요
- **Gemini API 키**: AI Studio 또는 GCP Console에서 발급된 키. Vertex AI 경로 미사용
- **Agit OAuth 토큰**: 어드민에서 모듈(그룹)별로 scope된 토큰 발급 가능 — 토큰 자체가 권한 경계
- **그룹 ID**: 각 모듈당 2개씩 하드코딩 (전자결재: 300019184, 300100971 / 인사: 300045170, 300102359)
