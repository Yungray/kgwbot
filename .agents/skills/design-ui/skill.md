# Design UI — Agit 챗봇 UI 설계 절차

CollaboAI Admin 디자인 시스템(`~/AGENTS.md`)을 기반으로 챗봇 화면을 설계한다.

## 워크플로우

### 1. 화면 목적·맥락 파악
- 누가 (운영팀? PM?), 언제 (CS 응대 중? 분석 보고 작성 중?), 무엇을 위해
- 기존 화면과의 관계 (사이드바 LNB의 어느 모듈? 어느 모드?)

### 2. 와이어프레임 정의
ASCII 또는 텍스트로 골격을 먼저 잡는다:
```
[Sidebar 248px] | [Topbar 64px]
                | [Mode tabs 44px]
                | [Welcome state 또는 Messages]
                | [Input area]
```

### 3. 컴포넌트 분해
- 재사용 가능한 단위로 분해 (모듈 카드, 모드 탭, suggest 카드, 메시지 버블 등)
- 각 컴포넌트에 대해 variant(예: primary/secondary, light/dark) 정의

### 4. 디자인 토큰 매핑
모든 색상·간격·타이포는 `~/AGENTS.md` 토큰 사용:
- 색상: `--indigo-500`, `--gray-50`, `--success-700` 등
- 타이포: 14px base, 11~32px 스케일, Inter 폰트
- 스페이싱: 8px base (xs 4 / sm 8 / md 12-16 / lg 24 / xl 32)

### 5. 상태 정의
모든 인터랙티브 요소에 5가지 상태:
- 기본 / 호버 / 포커스 / 액티브(선택됨) / 비활성

빈/로딩/에러 상태도 별도 정의.

### 6. 인터랙션 명세
- 클릭/키보드 단축키 동작
- 전환 애니메이션 (보통 0.15s ease)
- 한국어 IME 입력 처리 (Enter 두 번 전송 방지 등)

### 7. 코드 적용 가이드
- 영향 파일: `webapp/templates/index.html` / `webapp/static/style.css` / `webapp/static/app.js`
- 적용 순서: HTML 구조 → CSS 토큰 → JS 인터랙션

## 출력 규칙

- 화면 명세는 위 7단계를 모두 포함
- 토큰 미존재 색상이 필요하면 신규 토큰 제안 (이름 + hex + 용도)
- 모바일(<900px) 대응 방식 명시
- 기존 컴포넌트와 충돌하는 부분이 있으면 마이그레이션 영향 분리 기술

## 점검 체크리스트

- [ ] 5가지 상태 모두 정의됨
- [ ] 빈/로딩/에러 상태 정의됨
- [ ] 디자인 토큰 변수 사용 (하드 hex 없음)
- [ ] 한국어 카피 자연스러움
- [ ] 키보드 내비게이션 가능
- [ ] 색 대비 WCAG AA 이상
- [ ] 1440 데스크톱·900 모바일 모두 대응
