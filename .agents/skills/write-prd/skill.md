# Write PRD — Agit 챗봇 기능 명세 작성

기능 요청을 받아 구현 가능한 수준의 PRD/기능 명세로 변환한다.

## 워크플로우

### 1. 사용자·문제 정의
- Primary user: 운영팀(CS 응대), Secondary: PM/제품팀(VOC 분석)
- 해결할 문제 한 문장: "X 사용자가 Y 상황에서 Z를 못해 W가 일어남"
- 현재 우회/대안 (있다면): 수기로 어떻게 처리 중인지

### 2. 사용자 시나리오 작성
3~5개의 구체적 시나리오:
```
시나리오 A (정상 경로): 사용자가 → … → 결과 확인
시나리오 B (엣지 케이스): 데이터가 0건일 때 → …
시나리오 C (오류 경로): LLM 호출 실패 시 → …
```

### 3. User Story + Acceptance Criteria
```
### US-XXX: [기능명]
**As a** [역할]
**I want to** [할 일]
**So that** [얻고 싶은 가치]

**Acceptance Criteria:**
1. Given [상태] When [행위] Then [결과]
2. Given [엣지 케이스] When [행위] Then [기대 처리]

**의존성:** [선행 기능, 외부 API]
**우선순위:** Must / Should / Could / Won't
**측정 지표:** [수치 + 측정 방법]
```

### 4. 비기능 요구사항
- 성능: 응답 시간 목표 (예: LLM 호출 포함 10초 이내)
- 보안: 토큰 노출 방지, XSS, CSRF
- 가시성: 시작 로그, 도구 호출 chip, 에러 메시지

### 5. 우선순위·로드맵
- Now (이번 sprint), Next (다음 sprint), Later (백로그)
- 의존성 그래프

### 6. 측정·검증 계획
- 성공 지표 (KPI / 사용자 행동 지표)
- A/B 테스트 가능 여부
- 검증 방법 (수동 테스트 / 자동 회귀)

## 출력 규칙

- 모든 AC는 Given/When/Then 형식
- "사용자가 편하게" 같은 모호한 표현 금지 → "응답 시간 3초 이내", "도구 호출 chip 자동 표시" 등 측정 가능하게
- 모듈(전자결재/인사) 독립성 보장이 필요한 부분은 명시
- 기존 자산(`agit.py` 도구, `~/AGENTS.md` 토큰) 재사용 가능 여부 표기

## 점검 체크리스트

- [ ] 모든 US에 측정 지표 있음
- [ ] 엣지 케이스 시나리오 최소 1개
- [ ] 의존성·우선순위 표기
- [ ] 모듈 독립성 영향 평가
- [ ] 디자인·프롬프트 의뢰 사항 별도 분리
